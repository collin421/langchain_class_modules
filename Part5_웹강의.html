첫 미니 프로젝트
마케팅 카드 생성기

Part 1~4의 네 부품(모델·프롬프트·LCEL·파서)을 하나로 합쳐 작동하는 체인을 끝까지 만듦. 새 개념이 아니라 통합·실전이고, 마지막엔 "이 체인이 못 하는 것"을 마주하며 중급의 문을 엶.

🟢 초급 · 5/5 졸업 통합 프로젝트 예상 45분
5-1 · 5-2

설계가 먼저 — 입력과 출력

만들 것은 상품명만 넣으면 마케팅 카드가 나오는 체인임. 코드 전에 입력·출력을 못 박음 — 좋은 프로젝트의 90%는 "출력 구조를 어떻게 정의하느냐"에서 갈림.

인터랙티브 · 프로젝트 설계도

적은 입력(상품명 하나)에서 풍부한 출력(5개 필드)을 뽑는 구조임. Pydantic 모델이 곧 이 설계도가 됨.

입력
"베이지 케이블 니트"
(str)
→
출력 · MarketingCard
headline: str
description: str
target: str
hashtags: list[str]
price_range: str
📌 참고  필드명은 영어로(코드에서 다루기 쉬움), Field(description=...) 설명은 한국어로 쓰는 조합이 무난함. 설명이 구체적일수록 모델이 각 칸을 의도대로 채움.
3줄 요약
  1. 코드보다 입력·출력 설계를 먼저 못 박음.
  2. 출력 구조(Pydantic)가 곧 프로젝트의 설계도임.
  3. 적은 입력에서 풍부한 출력 = LLM 앱의 전형임.
5-3

체인 조립 — 부품을 배선하기

설계도가 있으니 프롬프트와 구조화 모델을 파이프로 이음. Part 2·3·4가 한 줄에 모임 — 이 한 줄이 프로젝트의 심장임.

인터랙티브 · 단계별로 조립하기

블록을 하나씩 추가하며 체인을 완성함. 마지막 블록까지 붙으면 카드가 배출됨.

1prompt — 마케터 역할 + 상품명 요청Part 2
2| model — gemini-3-flashPart 1
3.with_structured_output(MarketingCard)Part 4
아직 체인이 완성되지 않음 — 블록을 추가하세요
📌 참고  temperature는 마케팅 카피처럼 다양성이 필요하니 0.7 정도로 둠(Part 1). 매번 똑같이 뽑고 싶으면 0으로 내림. 체인은 한 번 조립해 두고 상품마다 .invoke()로 재사용함.
3줄 요약
  1. prompt | model.with_structured_output(...)가 심장임.
  2. 상품명이 프롬프트→모델을 거쳐 검증된 카드 객체로 나옴.
  3. 이 한 체인에 Part 2·3·4가 모두 녹아 있음.
5-4

확장 — 배치와 병렬

체인이 작동하면 Part 3의 .batch()와 RunnableParallel로 손쉽게 확장됨. 새 개념 없이 배운 도구의 조합만으로 프로젝트가 커짐.

인터랙티브 · 두 가지 확장
.batch() · 여러 상품
RunnableParallel · 여러 톤
📌 참고  결과가 객체라 card.model_dump()로 딕셔너리화해 스프레드시트·DB에 손쉽게 저장됨. 이런 "뒷단 연결"이 도구(Tool)와 에이전트를 배우는 중급에서 본격화됨.
3줄 요약
  1. .batch()로 여러 상품, RunnableParallel로 여러 버전을 한꺼번에.
  2. 새 개념 없이 Part 3 도구 조합만으로 확장됨.
  3. 객체라 저장·전달이 쉬워 자동화로 이어짐.
5-5

체인의 한계 — 직진 도로

우리 체인은 강력하지만 항상 정해진 길로만 흐름(Part 0의 "직진 도로"). 대부분의 작업엔 충분하지만, "상황을 보고 다음 행동을 그때그때 고르는" 일은 담을 수 없음.

인터랙티브 · 이 요구를 체인으로 풀 수 있을까

요구를 바꿔가며, 정해진 길(체인)로 풀 수 있는지 판정해봄.

📌 참고  체인이 부족해서 에이전트를 쓰는 게 아님. 순서가 고정된 일은 여전히 체인이 더 빠르고 안전함. 에이전트는 "동적 판단·도구·분기"가 필요한 순간에만 꺼내는 도구임. 이 경계 감각이 중급의 핵심임.
3줄 요약
  1. 체인은 강력하지만 항상 정해진 길로만 흐름.
  2. "검색할지 판단·분기"가 필요한 일은 체인으로 못 담음.
  3. 그때 필요한 것이 에이전트 — 중급의 시작임.
졸업 점검

스스로 점검 — 체인일까 에이전트일까

중급으로 가는 가장 중요한 감각은 "정해진 길인가, 동적 판단인가"임. 각 요구가 어느 쪽인지 골라봄.
