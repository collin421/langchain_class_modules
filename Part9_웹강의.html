🟡 중급 · 9/18 create_agent 예상 45분
9-1

create_agent — 세 가지 재료

1.0 에이전트의 정식 생성은 create_agent(model, tools, system_prompt)임. 신입에게 업무 환경을 세팅하듯, 역할(프롬프트)·장비(도구)·두뇌(모델)를 쥐여주면 스스로 일함. 내부는 LangGraph가 도구 호출 루프를 돌림.

from langchain.agents import create_agent

agent = create_agent(
    model=model,
    tools=[get_stock, search_policy],
    system_prompt="너는 리리마켓 상담원이야.",
)
인터랙티브 · 세 재료의 역할

각 재료를 눌러 무슨 역할인지 확인함.

🧠
model
두뇌 — 판단하고 답을 생성
🔧
tools
손발 — 검색·계산·조회 등 행동
📋
system_prompt
지침서 — 역할·규칙·도구 가이드
📌 참고  내부 루프: agent 노드가 모델을 부르고 → 응답에 tool_calls가 있으면 tools 노드가 실행해 ToolMessage로 추가 → 다시 모델 호출. tool_calls가 없을 때까지 반복 후 전체 messages 반환. 우리가 루프를 코딩하지 않음.
3줄 요약
  1. create_agent(model, tools, system_prompt)가 정식 생성임.
  2. 내부는 LangGraph가 "모델→도구→다시 모델"을 자동 반복함.
  3. tool_calls가 없어질 때까지 돌고 전체 messages를 반환함.
9-2

도구 조합이 곧 능력

에이전트의 능력은 쥐여준 도구 조합으로 정해짐 — 공구 벨트에 채운 연장들과 같음. 같은 에이전트가 질문 성격에 따라 다른 도구를 고름.

인터랙티브 · 질문 → 도구 라우팅

질문을 눌러, 4개 도구 중 에이전트가 무엇을 고르는지 봄.

📦
get_stock
💱
get_exchange_rate
🚚
track_delivery
📕
search_policy
3줄 요약
  1. 에이전트 능력은 쥐여준 도구 조합으로 정해짐.
  2. 같은 에이전트가 질문에 따라 다른 도구를 고름.
  3. 도구는 꼭 필요한 것을 명확한 설명과 함께(Part 7).
9-3

RAG를 도구로 — 필요할 때만 검색

Part 8의 RAG 체인을 @tool로 감싸면 "필요할 때만 문서를 찾는" 도구가 됨. 상담원 책상에 규정집을 놓아둔 것 — 언제 펼지는 에이전트의 판단임.

인터랙티브 · 규정집을 언제 펼까

질문 종류를 바꿔, 에이전트가 RAG 도구(규정집)를 펼치는지 일반 도구를 쓰는지 봄.

📦
get_stock
(재고)
📕
search_policy
(규정집·RAG)
📌 참고  체인(항상 검색)을 도구로 감싸면 "검색할지 말지"의 동적 판단이 에이전트의 몫이 됨 — Part 6의 "체인을 에이전트의 도구로 쓴다"가 실현됨. 체인의 안정성 + 에이전트의 유연함을 계층적으로 조합하는 핵심 패턴임.
3줄 요약
  1. RAG 체인을 @tool로 감싸면 "필요할 때만 검색"하는 도구가 됨.
  2. 검색 여부의 판단이 에이전트의 몫이 됨.
  3. 체인의 안정성 + 에이전트의 유연함을 계층 조합함.
9-4

system_prompt — 행동 지침서

system_prompt는 에이전트의 역할·규칙·도구 사용 가이드를 정함 — 신입 상담원에게 주는 근무 지침서임. 매 모델 호출 맨 앞에 붙어 모든 판단의 기준이 됨.

인터랙티브 · 지침을 켜고 끄며 행동 보기

규칙을 켜고 끄면, 그에 따라 에이전트의 행동이 어떻게 달라지는지 봄.

✓
친절하고 간결한 말투로 응대한다
✓
정책 질문은 반드시 search_policy 도구로 확인한다
✓
모르면 추측하지 말고 "확인이 어렵다"고 안내한다
📌 참고  str 또는 SystemMessage로 줌. docstring(도구 쪽 설명)과 system_prompt(에이전트 쪽 지침)가 함께 작동해 도구 선택을 좌우함. 결제·삭제 같은 위험 행동의 안전 규칙도 여기 명시함(Part 14 연결).
3줄 요약
  1. system_prompt는 역할·규칙·도구 가이드를 정함.
  2. 매 모델 호출 맨 앞에 붙어 모든 판단의 기준이 됨.
  3. docstring과 system_prompt가 함께 도구 선택을 좌우함.
9-5

복합 질문 — 여러 도구를 순차로

한 질문에 여러 정보가 필요하면, 에이전트가 루프를 돌며 도구를 여러 번 호출함. 실행 후 messages를 펼쳐 보면 그 흐름이 다 드러남(기본 디버깅).

인터랙티브 · messages 추적

"트렌치 코트 재고 확인하고, 반품 기간도 알려줘" — 두 도구가 순차로 불리는 과정을 단계별로 따라감.

📌 참고  입력 형식은 {"messages": [("user","...")]}. 결과는 전체 messages 리스트이며, 최종 답은 마지막 메시지임. 본격 관측·추적은 고급의 LangSmith(Part 16)에서 다룸.
3줄 요약
  1. 복합 질문이면 루프를 돌며 도구를 여러 번 순차 호출함.
  2. messages를 펼치면 도구 호출 흐름이 다 보임.
  3. 이 추적이 에이전트 기본 디버깅 도구임.
