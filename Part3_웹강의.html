LangChain 에이전트 마스터 · Part 3

LCEL 파이프(|)의 원리

지금까지 만든 두 블록(프롬프트·모델)은 모두 .invoke()를 가졌음. 그 공통점이 사실 약속(Runnable)이었음을 밝히고, 그 덕분에 블록을 파이프 하나로 줄줄이 잇는 법을 배움. 이 | 한 글자가 LangChain의 심장임.

Runnable과 파이프 — 컨베이어 벨트

LangChain의 거의 모든 부품은 Runnable이라는 공통 약속을 따름 — "나는 입력을 받아 출력을 내며 .invoke / .stream / .batch를 가진다." 표준 규격의 콘센트처럼, 같은 플러그를 쓰니 자유롭게 연결됨.

그 연결 문법이 파이프 |임. 왼쪽 블록의 출력을 오른쪽 블록의 입력으로 흘려보냄 — 컨베이어 벨트처럼.

chain = prompt | model

chain.invoke({"product": "베이지 니트"})
인터랙티브 · 데이터 흐름 애니메이션

"실행"을 누르면 입력이 벨트를 따라 각 공정을 통과하며 변환되는 과정을 단계별로 봄.

📌 참고  파이프로 이은 결과(chain)도 다시 Runnable임. 그래서 더 큰 체인의 한 블록으로 끼울 수 있음 — "조립한 것이 다시 부품이 되는" 성질이 복잡한 파이프라인을 단순하게 유지함.
3줄 요약
  1. Runnable은 모든 블록이 공유하는 약속(.invoke/.stream/.batch)임.
  2. 파이프 |는 왼쪽 출력을 오른쪽 입력으로 흘려보냄.
  3. 이은 체인도 다시 Runnable이라 더 큰 체인에 끼울 수 있음.
3-3

체인 전체에 invoke · stream · batch

체인도 다시 Runnable이라, 개별 블록에 쓰던 실행 메서드가 조립된 파이프라인 전체에 그대로 작동함.

인터랙티브 · 같은 체인, 세 가지 실행

chain = prompt | model에 세 메서드를 각각 걸면 결과가 어떻게 도착하는지 비교함.

.invoke()
.stream()
.batch()
📌 참고  .stream()은 스트리밍을 지원하는 마지막 블록(모델)까지 자동 전파됨. 비동기 버전(.ainvoke 등)도 체인 전체에 동일하게 적용됨.
3줄 요약
  1. 체인 전체가 Runnable이라 실행 메서드가 통째로 적용됨.
  2. .stream()은 모델부터 조각을 실시간으로 흘려보냄.
  3. 챗봇은 stream, 대량 처리는 batch가 정석임.
3-4

Parallel · Passthrough — 흐름을 나누기

체인은 한 줄로만 흐르지 않음. RunnablePassthrough는 입력을 가공 없이 그대로 통과시키고, RunnableParallel은 여러 블록을 동시에 실행해 결과를 딕셔너리로 모음.

인터랙티브 · 병렬 분배기

한 입력 "울 코트"를 세 갈래로 동시에 처리함. "병렬 실행"을 눌러 각 갈래가 한꺼번에 작동하는 걸 봄.

입력: "울 코트"
↙↓↘
original
(Passthrough)
감성
(체인)
실용
(체인)
→ 결과를 기다리는 중...
📌 참고  RunnablePassthrough.assign(...)은 "기존 입력 유지 + 새 키 추가"임. RAG 체인에서 "원래 질문 + 검색된 문서"를 함께 넘길 때 자주 등장함(Part 8).
3줄 요약
  1. RunnablePassthrough는 입력을 그대로 통과시킴.
  2. RunnableParallel은 여러 블록을 병렬 실행해 딕셔너리로 모음.
  3. 이 둘은 RAG 체인(질문 그대로 + 동시에 문서 검색)의 핵심 패턴임.
3-5

RunnableLambda — 내 함수를 체인에

체인 중간에 내가 만든 평범한 파이썬 함수를 끼우고 싶을 때 RunnableLambda가 그 함수를 Runnable로 감싸 줌. 파이프 안의 함수·람다는 자동으로 변환되기도 함.

인터랙티브 · 변환 어댑터

내 함수 shout는 원래 LangChain 규격이 아님. "어댑터 끼우기"를 누르면 Runnable로 감싸져 파이프에 꽂힘.

"hello"
입력
|
shout()
일반 함수 (규격 ✗)
→
?
출력
어댑터를 끼우기 전에는 파이프에 꽂을 수 없음.
📌 참고  다음 파트의 출력 파서(StrOutputParser 등)도 사실 "모델 출력을 가공하는 Runnable"의 잘 만들어진 버전임. 간단한 가공은 RunnableLambda, 정형 파싱은 전용 파서를 씀.
3줄 요약
  1. RunnableLambda는 내 일반 함수를 Runnable로 감쌈.
  2. 파이프 안의 함수·람다는 자동으로 변환됨.
  3. 간단한 후처리에 유용, 정형 파싱은 전용 파서를 씀.
