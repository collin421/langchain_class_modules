프롬프트 템플릿

형식은 똑같고 내용만 바뀌는 질문을 매번 손으로 쓰는 대신, 빈칸이 뚫린 틀을 한 번 만들어 변수만 갈아 끼워 찍어냄. 이 "틀 + 변수" 구조가 다음 파트의 LCEL 파이프로 이어지는 다리임.

🟢 초급 · 2/5 PromptTemplate 예상 35분
2-1 · 2-2

왜 템플릿인가 — 빈칸 양식지

같은 작업을 여러 입력에 반복할 때 문자열을 +로 직접 조립하면 금세 지저분해짐. 프롬프트 템플릿은 빈칸이 뚫린 양식지임 — 양식은 한 번 만들고, 빈칸만 바꿔 끼움.

from langchain_core.prompts import PromptTemplate

template = PromptTemplate.from_template(
    "다음 상품을 {tone} 톤으로 소개해줘: {product}"
)
template.invoke({"product": "베이지 니트", "tone": "감성적인"})

from_template은 도장 새기기임. 도장을 한 번 새겨 두면 종이마다 빈칸만 채워 무한히 찍어낼 수 있음.

인터랙티브 · 도장 찍기

아래 빈칸 값을 바꾸면 템플릿의 슬롯이 실시간으로 채워짐. 양식은 그대로, 내용만 바뀌는 걸 봄.

↓ .invoke()
3줄 요약
  1. 문자열 직접 조립은 반복·수정·실수에 약함.
  2. 템플릿은 빈칸 양식지 — 변수만 갈아 끼움.
  3. from_template + .invoke({...})로 찍어냄.
2-3

ChatPromptTemplate — 메시지 템플릿

채팅 모델은 역할이 붙은 메시지로 대화함. ChatPromptTemplate은 System/Human 역할을 살리면서 변수를 끼움. ("system", "...") 같은 (역할, 템플릿) 튜플로 지정함.

인터랙티브 · 두 템플릿의 결과 비교

같은 내용을 두 템플릿으로 만들면 결과물이 어떻게 다른지 봄. 채팅 모델엔 오른쪽(메시지 리스트)이 기본임.

📌 참고  대화 기록을 통째로 끼울 자리가 필요하면 MessagesPlaceholder를 씀(메모리·Part 10에서 재등장). 채팅 모델 실무는 거의 ChatPromptTemplate이 기본임.
3줄 요약
  1. ChatPromptTemplate은 역할이 붙은 메시지를 템플릿으로 묶음.
  2. ("system","...") 튜플로 역할+템플릿을 지정함.
  3. 결과는 ChatModel에 그대로 넣는 메시지 리스트임.
2-4

변수 주입 — partial로 미리 고정

템플릿은 input_variables로 자신의 빈칸을 알고 있음. partial()로 일부 변수를 미리 고정하면, 남은 변수만 나중에 채우면 됨.

partial은 도장에 미리 새겨 둔 회사 로고와 같음. 로고(브랜드명)는 이미 박혀 있고, 가변 정보(상품명)만 그때그때 채움.

인터랙티브 · 변수 고정하기

템플릿 "{brand}의 {product} 문구를 써줘". 변수를 눌러 고정/해제하며 남은 빈칸이 어떻게 줄어드는지 봄.

brand product
고정된 변수 (partial)
없음
호출 시 채워야 할 빈칸 (input_variables)
📌 참고  partial에는 값 대신 함수도 넣을 수 있음. "현재 시각"처럼 호출 때마다 달라져야 하는 값은 함수를 걸어 매번 새로 계산되게 함.
3줄 요약
  1. 템플릿은 input_variables로 빈칸 목록을 앎.
  2. .invoke() 키가 목록과 맞아야 하며, 누락 시 오류남.
  3. partial로 일부 변수(값·함수)를 미리 고정함.
2-5

Few-shot — 예시로 형식 가르치기

지시만 주는 zero-shot 대신, 잘 된 예시 몇 개를 보여주는 few-shot은 출력 형식·스타일을 훨씬 잘 따르게 만듦. 신입에게 "이렇게 쓰면 돼"라며 견본을 보여주는 것과 같음.

인터랙티브 · 예시를 켜고 끄며 비교

예시를 켜고 끄면 완성 프롬프트가 어떻게 바뀌는지, 그리고 형식 통제력이 어떻게 달라지는지 봄.

📌 참고  채팅 모델용은 FewShotChatMessagePromptTemplate, 예시가 많으면 ExampleSelector로 입력과 비슷한 예시를 자동 선별함. 예시는 질 좋은 2~5개가 양보다 중요함.
3줄 요약
  1. few-shot은 예시로 출력 형식·스타일을 가르침.
  2. FewShotPromptTemplate = 예시 틀 + 예시 데이터 + suffix.
  3. 잘 고른 2~5개가 형식 통제에 효과적임.
자가 점검

가장 흔한 에러 — 변수 누락

초급자가 가장 자주 만나는 오류는 "채워야 할 빈칸을 빠뜨린" 경우임. input_variables를 떠올리며 어떤 키를 넣어야 하는지 직접 맞춰봄.

인터랙티브 · 빈칸 채우기 점검

아래 템플릿을 호출하려면 어떤 키가 필요할까? 키를 골라 켜고 "호출"이 성공하는지 봄.

"{brand}의 {product}를 {tone} 톤으로 소개해줘"
brand product tone color
핵심 정리
  1. 호출 딕셔너리의 키 = 템플릿의 input_variables (정확히 일치).
  2. 빠뜨리면 "변수 누락" 오류, 불필요한 키는 무시되거나 경고.
  3. 막히면 template.input_variables를 찍어 확인하면 됨.
